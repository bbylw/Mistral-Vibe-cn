快速入门

本指南将帮助您快速上手 Vibe。更高级的自定义请参考 配置指南。

开启您的第一个项目

首先，进入您的项目根目录：

cd /path/to/your/project

然后启动 Vibe：

vibe

首次运行说明

  • 自动创建默认配置：~/.vibe/config.toml
  • 提示您选择界面主题。
  • 提示您输入 API 密钥（如果未配置）。可以从 Mistral Console 获取。

交互模式

直接运行 vibe 即可进入与 AI 代理的对话循环。这是推荐的使用方式。

多行输入

使用 Ctrl+J 或 Shift+Enter 插入换行符。

引用文件

在提示中使用 @ 符号触发智能补全，例如：> 读取文件 @src/main.js

执行 Shell

使用 ! 前缀直接执行命令，绕过 AI：> !ls -l

常用命令与快捷键

📐 斜杠命令

  • /h, /help: 显示帮助信息
  • /stats, /status: 查看代理统计数据
  • /cfg, /config, /model, /theme: 编辑配置或切换模型
  • /r, /reload: 重新加载磁盘配置
  • /clear, /reset: 清除对话历史
  • /log, /logpath: 显示当前交互日志文件路径
  • /compact, /summarize: 总结并压缩对话历史
  • /exit, /q, /quit: 退出程序

⌨️ 键盘快捷键

  • Enter: 发送消息
  • Ctrl+J / Shift+Enter: 插入换行
  • Shift+Tab: 切换自动批准模式 (Auto-approve)
  • Ctrl+O: 切换工具输出视图显示
  • Ctrl+T: 切换待办事项 (Todo) 视图显示
  • Ctrl+C: 退出（或清除当前输入）
  • Escape: 中断代理或关闭对话框

🚀 自动批准模式 (Auto-approve)

使用 --auto-approve 标志启动 Vibe 可以自动批准所有工具执行，无需手动确认。

在交互模式下，您随时可以通过按 Shift+Tab 快捷键来切换此模式的开启/关闭状态。

编程模式

您也可以通过 --prompt 标志或管道输入非交互式地运行 Vibe，这非常适合脚本自动化：

vibe --prompt "重构 cli/main.py 中的主函数使其更模块化。"

注意：在编程模式下，Vibe 默认使用 自动批准模式 (auto-approve)。

安全防护与确认

对于实质性的任务（重构、多文件更新），Vibe 会将工作拆分为多个安全步骤，并在执行前请求您的确认。

运行完成后，您可以查看到完整的执行日志和输出结果。
